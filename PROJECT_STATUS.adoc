This document is the current status snapshot for the Boundary framework monorepo.

== Executive Summary

Boundary has shipped `v1.0.0-alpha` and is operating as a 13-library monorepo with consistent Functional Core / Imperative Shell architecture.

The project is in a strong pre-GA state:

* Core platform capabilities are implemented (auth, admin UI, storage, jobs, cache, email, realtime, tenant support, scaffolder).
* CI is comprehensive and includes per-library tests, whole-suite tests, docs linting, and AGENTS link validation.
* Documentation and developer workflows are actively maintained.

Primary remaining maturity items are focused on finishing currently flagged in-development/roadmap areas, not on foundational architecture.

== Delivery and Quality Signals

=== CI/Automation

The main CI workflow (`.github/workflows/ci.yml`) currently includes:

* `lint` (clj-kondo)
* `docs-lint`
* `agents-links` (`python3 scripts/check-agents-links.py`)
* Per-library test jobs (`:core`, `:observability`, `:platform`, `:user`, `:admin`, `:storage`, `:cache`, `:jobs`, `:tenant`, `:email`, `:realtime`, `:scaffolder`)
* Whole-suite test job (`clojure -M:test:db/h2`)
* `test-summary` gate

=== Documentation and Agent Tooling

Recent documentation/process improvements include:

* Streamlined root `AGENTS.md` with reduced duplication.
* Standardized section template across all module `libs/*/AGENTS.md` files.
* Dedicated AGENTS link checker script and CI enforcement.
* Repo-native pre-commit hook path (`.githooks`) via `scripts/install-git-hooks.sh`.

== Known Gaps / Active Risks

* `libs/external` remains explicitly in-development and should not be treated as production-ready.
* Realtime module documentation still calls out future multi-server enhancements (Redis-backed registry/pub-sub).
* Some historical status documents in `docs/archive/` are stale and should not be used as operational source-of-truth.

== Recommended Priorities (Next 30 Days)

. Close `boundary-external` scope definition and ship first production-capable adapter set.
. Decide realtime scaling path (single-node hardening vs Redis-backed multi-node support) and align tests/docs.
. Continue concise modernization for remaining long architecture/reference pages while keeping `docs-lint` at zero warnings.
. Keep CI signal quality high by preserving fast per-library feedback and enforcing docs/link hygiene.
