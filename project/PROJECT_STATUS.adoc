:revdate: 2026-04-15

Current status snapshot for the Boundary framework monorepo.

== Summary

Boundary is at `v1.0.1-alpha-12` and operates as a 22-library monorepo (plus `boundary-tools`) with consistent Functional Core / Imperative Shell architecture.
All core platform capabilities are implemented. CI is comprehensive (e2e job temporarily disabled post alpha-12 to reduce pipeline time; runnable on demand). Documentation is actively maintained.
Roadmap Phases 1-2 (workflow, search, reports, calendar) are complete. Geo (Phase 4) was delivered ahead of schedule. Payments library is publish-ready. End-to-end testing (Playwright/spel) added for login sequence and admin UI. Six automated quality gates enforce FC/IS, placeholder tests, and dependency rules. Post-release work has focused on follow-up FC/IS refactors in `boundary-core` and hardening the `boundary-tools` deprecated-wrapper scanner (BOU-15). Phase 3 (push, audience) is the next priority.

== CI / Automation

The main CI workflow (`.github/workflows/ci.yml`) runs:

* `lint` — clj-kondo
* `docs-lint` — documentation drift linter
* `agents-links` — `bb scripts/check_agents_links.clj`
* Per-library test jobs for all 22 libraries
* Whole-suite test job (`clojure -M:test:db/h2`)
* `test-summary` gate
* `e2e` — defined (Playwright/spel) but currently gated `if: false`; run manually via `bb e2e`

== Known Gaps

* `boundary-external` is explicitly in-development and not production-ready.
* `boundary-realtime` multi-server scaling (Redis-backed registry/pub-sub) is still roadmap.
* Roadmap Phase 3 libraries (`boundary-push`, `boundary-audience`) not yet implemented.
* Roadmap Phase 4 library `boundary-forms` not yet implemented.
* `e2e` CI job is disabled (`if: false`) and must be re-enabled once pipeline-time budget allows.
* Some historical documents in `docs/archive/` are stale and should not be used as operational source-of-truth.

== Recommended Priorities

. Start Roadmap Phase 3: implement `boundary-push` (21d) and `boundary-audience` (18d).
. Re-enable `e2e` CI job (remove `if: false`) once runtime budget is acceptable; alternatively schedule it nightly instead of per-PR.
. Validate `boundary-external` adapters against live Twilio sandboxes and mark production-ready.
. Decide realtime scaling path (single-node hardening vs Redis-backed) and align tests/docs.
. Publish `v1.0.1-alpha-12` to Clojars (all 23 artifacts including payments).
. Keep CI signal quality high: fast per-library feedback and enforced docs/link hygiene.
