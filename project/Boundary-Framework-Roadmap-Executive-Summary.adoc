== Strategic Context

=== Opportunity

Analysis of two production Django applications (WBA and OTTO) revealed **8 strategic gaps** in the Boundary Framework that, once filled, will significantly accelerate project delivery and reduce costs across both B2B and B2C domains.

*Key Insight*: These libraries complement each other perfectly - WBA provides B2B enterprise patterns (workflows, search, reporting), while OTTO provides B2C/mobile patterns (push notifications, audience targeting, forms).

== Implementation Phases

=== Phase 1: Foundation (Q2 2026) - *€26.400*

*Goal*: Build must-have B2B capabilities with fastest ROI

*Deliverables*:

- ✅ boundary-workflow - State machines and business process automation
- ✅ boundary-search - Full-text search (PostgreSQL + Elasticsearch)

*Timeline*: 8 weeks (33 development days)

*Expected Impact*:

- 80% B2B project adoption
- €15k-€22k savings per project
- Breakeven after 2 projects

*Decision Gate*: Project adoption ≥60% + 2 successful implementations → GO to Phase 2

---

=== Phase 2: Enhanced Productivity (Q3 2026) - *€21.600*

*Goal*: Add reporting and scheduling capabilities

*Deliverables*:

- ✅ boundary-reports - PDF/Excel/Word (DOCX) generation and templates
- ✅ boundary-calendar - Event scheduling and availability management

*Timeline*: 7 weeks (27 development days)

*Expected Impact*:

- 50-60% project adoption
- €6k-€9k additional savings per project
- Cumulative savings: €21k-€31k per project

*Decision Gate*: Reports used in 40%+ projects + Calendar used in 30%+ projects → GO to Phase 3

---

=== Phase 3: B2C Capabilities (Q4 2026) - *€31.200*

*Goal*: Enable mobile/consumer-facing applications

*Deliverables*:

- ✅ boundary-push - Mobile push notifications (FCM/APNs)
- ✅ boundary-audience - User segmentation and targeting

*Timeline*: 10 weeks (39 development days)

*Expected Impact*:

- 40-60% B2C project adoption
- €7k-€11k additional savings per B2C project
- Total portfolio: 6 production-ready libraries

*Decision Gate*: Push used in 40%+ B2C projects + Positive stakeholder feedback → EVALUATE Phase 4

---

=== Phase 4: Specialized Features (Q1 2027) - *€23.200* [CONDITIONAL]

*Goal*: Add nice-to-have specialized capabilities

*Deliverables*:

- ⚠️ boundary-geo - Geocoding and mapping (if demand exists)
- ⚠️ boundary-forms - Dynamic form builder (if not replaced by external service)

*Timeline*: 7 weeks (29 development days)

*Expected Impact*:

- 20-40% project adoption
- €3k-€5k additional savings per project
- Portfolio completion

*Decision Gate*: Evaluate Q4 2026 usage patterns + market demand → GO/PIVOT/CANCEL

== Recommendation

=== Primary Recommendation: *GO - Phases 1-3*

*Rationale*:

1. *Strong ROI*: €79.2k investment → €865k return over 5 years (1093%)
2. *Fast Breakeven*: 3-4 projects (7-12 months)
3. *High Confidence*: Critical gaps identified from real production apps
4. *Balanced Risk*: Phase 4 optional buffer based on actual adoption
5. *Strategic Alignment*: Covers both B2B and B2C project needs

*Investment*: €79.200 (99 development days) over 3 quarters (Q2-Q4 2026)

*Expected Return*: €21k-€31k savings per project × 45 projects = €945k over 5 years

---

=== Phase 4: *CONDITIONAL - Evaluate Q4 2026*

*Rationale*:

1. *Lower Priority*: Geo and forms have 20-40% adoption vs 50-80% for Phases 1-3
2. *Market Alternatives*: Services like Google Maps API and Typeform may be sufficient
3. *Opportunity Cost*: €23.2k could be invested in higher-priority features based on actual usage patterns
4. *Risk Management*: Allows us to stop at €79.2k if adoption is lower than expected

*Decision Point*: Q4 2026 after evaluating Phases 1-3 adoption and demand for geo/forms features.

== Appendix: Library Quick Reference

=== boundary-workflow (Priority 1)

*Problem*: Every B2B app needs state machines (approvals, orders, tickets) - currently implemented ad-hoc

*Solution*: Declarative FSM with guards, callbacks, persistence, audit trails, parallel states

*ROI*: €8k-€12k per project (saves 10-15 days of custom state machine development)

---

=== boundary-search (Priority 2)

*Problem*: Search functionality requires complex implementation (PostgreSQL FTS or Elasticsearch integration)

*Solution*: Unified search API with PostgreSQL FTS (default) and Elasticsearch adapter (optional)

*ROI*: €7k-€10k per project + €5k/year operational savings (avoid Elasticsearch Cloud if unnecessary)

---

=== boundary-reports (Priority 3)

*Problem*: PDF/Excel generation requires low-level libraries with poor ergonomics

*Solution*: Template-based report generation (PDF, Excel, Word/DOCX) with Hiccup-style DSL

*ROI*: €3.2k-€4.8k per project (saves 4-6 days of report generation code)

---

=== boundary-calendar (Priority 4)

*Problem*: Event scheduling, availability, recurring events require complex date/time logic

*Solution*: iCal-compatible calendar with availability checking, conflict detection, recurring events

*ROI*: €3.2k-€4k per project (saves 4-5 days of calendar implementation)

---

=== boundary-push (Priority 5)

*Problem*: Mobile push notifications require FCM/APNs integration with retry logic, queuing, analytics

*Solution*: Unified push API with FCM/APNs adapters, delivery tracking, segmentation, scheduling

*ROI*: €4k-€6k per project (saves 5-8 days + €2k/year operational costs vs OneSignal)

---

=== boundary-audience (Priority 6)

*Problem*: User segmentation for campaigns requires complex querying and targeting logic

*Solution*: Segment builder with rule engine, dynamic queries, A/B testing support

*ROI*: €3k-€5k per project (saves 4-6 days of segmentation logic)

---

=== boundary-geo (Priority 7)

*Problem*: Geocoding and mapping features require third-party API integration

*Solution*: Geocoding abstraction with Google Maps/Nominatim adapters, caching, batch operations

*ROI*: €400-€800 per project (saves 0.5-1 day of integration code)

---

=== boundary-forms (Priority 8)

*Problem*: Dynamic form builders for surveys/applications require complex state management

*Solution*: Form builder with conditional logic, validation, versioning, response storage

*ROI*: €2.5k-€4k per project (saves 3-5 days vs custom forms or Typeform integration)
