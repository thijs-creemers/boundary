= Boundary at a Glance
:description: One-page overview of Boundary Framework

[.lead]
Production-ready Clojure with enforced Functional Core / Imperative Shell boundaries.

== In 30 Seconds

[source,text]
----
Boundary = module-first monorepo + FC/IS architecture

Inside each module:
- core/   : pure business logic
- ports   : contracts/protocols
- shell/  : HTTP/CLI/DB/external I/O
----

== Why Teams Use It

* Predictable architecture with strict dependency flow
* Fast, isolated tests for pure core logic
* Built-in modules for auth, admin, storage, jobs, search, workflow, tenant, and more
* Clear path from modular monolith to extracted services

== Quick Start

[source,bash]
----
git clone https://github.com/thijs-creemers/boundary
cd boundary
export JWT_SECRET="your-secret-minimum-32-characters-long"
clojure -M:repl-clj
----

[source,clojure]
----
(require '[integrant.repl :as ig-repl])
(ig-repl/go) ; starts app (default: http://localhost:3000)
----

== Core Commands

[cols="1,2"]
|===
|Command |Purpose

|`clojure -M:test:db/h2`
|Run all tests

|`clojure -M:test:db/h2 :user`
|Run one library test suite

|`clojure -M:clj-kondo --lint src test libs/*/src libs/*/test`
|Lint project code

|`clojure -M:migrate up`
|Run DB migrations

|`clojure -M:docs-lint`
|Validate docs namespace/path/link references
|===

== Testing Model

* Unit: pure core behavior
* Integration: shell orchestration
* Contract: adapter/DB behavior

== Good Fit

* Medium to large business domains
* Teams that need long-term maintainability
* Systems requiring multiple interfaces (HTTP/CLI/Web)
