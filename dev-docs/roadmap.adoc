= Boundary Framework Roadmap
:toc:
:toclevels: 3
:icons: font

Boundary is a module-centric Clojure framework implementing Functional Core / Imperative Shell. This roadmap tracks progress toward 1.0 and beyond. Updated April 2026.

== Phase 0 — Alpha icon:check-circle[] *COMPLETE*
*Goal: Make Boundary usable for early adopters with minimal friction.*

=== Delivered
- [x] Core APIs frozen (FC/IS, Interceptors, Normalized Routing)
- [x] All modules use the normalized routing system
- [x] Scaffolder generates normalized routes, interceptor hooks, CRUD stubs
- [x] Beginner documentation: Getting Started, Your First Module, FC/IS guide
- [x] Public release: GitHub open, Clojars deployment (`bb deploy --all`)
- [x] 22 independently publishable libraries under `libs/` (+ `boundary-tools`)
- [x] Example application: `examples/ecommerce-api/`

=== Test Health

As of April 2026: **3044 tests, 15742 assertions, 0 failures** across all 22 libraries (`clojure -M:test:db/h2`). CI runs per-library jobs plus a whole-suite gate, lint, docs-lint, and AGENTS link checks on every push.

== Strategic Notes

=== 1. Focus on onboarding, not architectural depth
Boundary already has deep documentation. The guided walkthrough (`boundary-starter/WALKTHROUGH.adoc`) now provides a narrative, 10-minute experience for beginners.

=== 2. Use the Scaffolder as your flagship feature
Boundary's scaffolder (`bb scaffold`) is a key differentiator — it should define the developer workflow from day one. With NL scaffolding (`bb scaffold ai`) and module integration (`bb scaffold integrate`), the end-to-end module creation flow is now nearly automated.

=== 3. Developer tooling reduces friction
Config Doctor (`bb doctor`), Setup Wizard (`bb setup`), and the AI tools (`bb ai`) address the biggest friction points for new developers: configuration, debugging, and boilerplate.

=== 4. Publish early, iterate publicly
The framework is already more mature than most early-stage OSS releases. Early feedback accelerates correctness.

=== 5. First-class internationalisation
With the admin UI fully translatable via `[:t :key]` markers in entity configs and catalogues shipped for English and Dutch, Boundary is ready for non-English-speaking markets out of the box. New modules inherit i18n support automatically.
