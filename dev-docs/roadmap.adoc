= Boundary Framework Roadmap
:toc:
:toclevels: 3
:icons: font

Boundary is publicly released and growing. This roadmap tracks progress from initial release through ecosystem maturity. Phases 0 and 1 are complete; Phase 2 (v1.0) is in progress; Phase 3 (Ecosystem) is running ahead of schedule — most ecosystem libraries were built before v1.0 is finalised.

*Last updated: 2026-03-21*

'''

== Phase 0 — Alpha ✅ COMPLETE
*Goal: Make Boundary usable for early adopters with minimal friction.*

=== Objectives
- ✅ Freeze core APIs (FC/IS, Interceptors, Normalized Routing).
- ✅ Provide a complete beginner-focused developer journey.
- ✅ Publish the framework publicly for early feedback.

=== Deliverables
1. ✅ All modules normalised to the new routing system (ADR-008, ADR-009).
2. ✅ Scaffolder generates:
   - normalised routes
   - interceptor hooks
   - minimal CRUD stubs
3. ✅ Example project (`boundary-examples`) with ecommerce API, user and inventory modules, login flow and CRUD.
4. ✅ Minimal documentation set:
   - *AGENTS.md* comprehensive developer guide
   - Per-library *AGENTS.md* guides for all 20 libraries
   - *CLAUDE.md* project instructions
5. ✅ Public release:
   - GitHub repository open
   - Clojars deployment pipeline (`bb deploy --all`)
   - Announcement drafts prepared (Clojurian Slack, Reddit, ClojureVerse)

'''

== Phase 1 — Beta ✅ COMPLETE
*Goal: Make Boundary comfortable and delightful to build real-world applications.*

=== Objectives
- ✅ Improve day-to-day development experience.
- ✅ Provide practical, real-world examples.
- ✅ Strengthen web UI and CLI tooling.

=== Deliverables
1. ✅ CRUD Web UI for the User module (admin module + UI overhaul, `ui-style` library).
2. ✅ Example Interceptors via ADR-010 multi-layer interceptor pattern:
   - Authentication
   - Audit logging
   - Redis-backed rate limiting
3. ✅ Scaffolder improvements:
   - Optional UI generation flags
   - REST + CLI enhancements
   - AI-assisted scaffolding (`bb scaffold ai "<description>"`)
   - Interactive wizard (`bb scaffold`)
4. ✅ Example applications: `boundary-examples` repository with ecommerce API.
5. ⬜ Documentation expansion (partially done — more needed before v1.0):
   - Error handling guide
   - Observability quickstart
   - CI/CD templates (Docker done; Fly.io / Heroku pending)
6. ✅ Admin UX patterns:
   - Forms, tables, pagination, layout guidance
   - `admin-pilot` design system via `boundary-ui-style`

=== Notes
Developers can now build full production CRUD applications with admin UI, authentication, background jobs, caching, file storage, real-time messaging, and multi-tenancy — all scaffolded from the CLI.

== Phase 3 — Ecosystem Expansion 🚀 AHEAD OF SCHEDULE
*Goal: Build a vibrant Boundary ecosystem and optional advanced capabilities.*

NOTE: Most ecosystem libraries were completed before v1.0, driven by real project needs. The SaaS-level integration story is the remaining focus.

=== Delivered (ahead of schedule)

- ✅ `boundary-reports` — `defreport` macro, PDF/CSV export, scheduler integration.
- ✅ `boundary-calendar` — `defevent` macro, RRULE recurrence, iCal import/export, conflict detection, Hiccup views.

- ✅ `boundary-ai` — multi-provider AI (Ollama / Anthropic / OpenAI), NL scaffolding, error explainer, test generator, SQL copilot, documentation wizard.

== Strategic Notes

=== 1. Scaffolder is the flagship feature
`bb scaffold` and `bb scaffold ai` define the Boundary developer workflow. Keep investing here.

=== 2. Ecosystem arrived early — align docs and examples
All 20 libraries exist, but the docs and starter templates have not caught up. The v1.0 push is primarily about documentation, deployment guidance, and integration examples — not new library code.

=== 3. SaaS Starter Kit is the key adoption unlock
A pre-assembled SaaS template (login + storage + admin + tenant + billing hooks) would dramatically lower the barrier for teams evaluating Boundary for new projects.

=== 4. Publish early, iterate publicly
Early release accelerated correctness. Continue shipping small, frequently, with real-world usage driving priorities.
